+ (void) beforePaint(callback)

Note: Given method will be called with `complete` function as parameter. As soon as your preparations are done you should call that function.

Sets the method which will controll the painting preparation proccess.

This method will be called right ater previous page Joosy.Page.erase and in parallel with page data fetching so you can use it to initiate preloader.

Examples:

Sample before painter 

@beforePaint (complete) ->
  if !@data # checks if parallel fetching finished
    $('preloader').slideDown -> complete()

+ (void) paint(callback)

Note: Given method will be called with `complete` function as parameter. As soon as your preparations are done you should call that function.

Sets the method which will controll the painting proccess.

This method will be called after fetching, erasing and beforePaint is complete. It should be used to setup appearance effects of page.

Examples:

Sample painter 

@paint (complete) ->
  @container.fadeIn -> complete()

+ (void) erase(callback)

Note: Given method will be called with `complete` function as parameter. As soon as your preparations are done you should call that function.
Note: This method will be caled _before_ unload routines so in theory you can access page data from that. Think twice if you are doing it right though.

Sets the method which will controll the erasing proccess.

Use this method to setup hiding effect.

Examples:

Sample eraser 

@erase (complete) ->
  @container.fadeOut -> complete()

+ (void) fetch(callback)

Note: Given method will be called with `complete` function as parameter. As soon as your preparations are done you should call that function.

Sets the method which will controll the data fetching proccess.

Examples:

Basic usage 

@fetch (complete) ->
  $.get '/rumbas', (@data) => complete()

+ (void) fetchSynchronized(callback)

Note: This will work through {Joosy.Modules.Events#synchronize}

Sets the several separate methods that will fetch data in parallel.

Examples:

Basic usage 

@fetchSynchronized (context) ->
  context.do (done) ->
    $.get '/rumbas', (data) =>
      @data.rumbas = data
      done()
  
  context.do (done) ->
    $.get '/kutuzkas', (data) =>
      @data.kutuzkas = data
      done()

+ (void) scroll(element, options = {})

Note: If you use animated scroll joosy will atempt to temporarily fix the height of your document while scrolling to prevent jump effect.

Sets the position where page will be scrolled to after load.

Parameters:

  • element (jQuery) — Element to scroll to
  • options (Hash)

Options Hash: (options):

  • speed (Integer) — Sets the animation duration (500 is default)
  • margin (Integer) — Defines the margin from element position. Can be negative.

- (void) __callSyncedThrough(entity, receiver, params, callback)

Note: Magic People Voodoo People

Proxies callback through possible async wrapper.

If wrapper is defined, it will be called with given callback as one of parameters. If wrapper is not defined callback will be called directly.

Parameters:

  • entity (Object) — Object possibly containing wrapper method
  • receiver (String) — String name of wrapper method inside entity
  • params (Hash) — Params to send to wrapper, callback will be attached as the last of them.
  • callback (Function) — Callback to run
